## 1. Трассировка маршрута с использованием ICMP

Требует raw-сокета $\Rightarrow$ запускать через `sudo`.

```bash
sudo ./bin/traceroute <host> [--queries 3] [--max-hops 30] [--timeout 2s] [--resolve]
```

- `--queries` -- число пакетов на каждый маршрутизатор, для каждого выводится RTT
- `--resolve` -- задание Б

## 2. Дистанционно-векторная маршрутизация

```bash
./bin/dvrouting --mode sync
```
```bash
./bin/dvrouting --mode async
```

- Алгоритм Беллмана-Форда
- `--mode async` -- задание В: каждый узел это отдельная горутина, у узла своя очередь (буферизованный канал), соседи асинхронно пишут в неё векторы; \
сходимость определяется атомарным счётчиком сообщений
- Юнит-тесты проверяют корректность таблиц и их пересчёт при изменении стоимости канала для обоих режимов.

## 3. Использование протокола IPv6

```bash
./bin/echo6-server --addr <addr>
```
```bash
./bin/echo6-client --addr <addr> --message 'hello, ipv6'
```

## 4. Удаленное рисование

```bash
./bin/draw-server --addr <addr>
```
```bash
./bin/draw-client --addr <addr> [--r 0 --g 0 --b 0]
```

- Каждый сегмент штриха передаётся по TCP
- Запустите сервер, затем клиента. Клиент не будет работать без сервера. 
- `--r/--g/--b` задают цвет штриха.
